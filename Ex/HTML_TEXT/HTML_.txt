■1■ HTML이 무엇인가?
------------------------------------------------------------------------
●[웹브라우저]라는 SW에서 [문자열], [이미지], [동영상], [입력양식] 등을 출력하기 위해 사용되는 언어를 말한다.
●HyperText Markup Language 의 약어이다.
●HTML 은 실무적으로 [웹브라우저]가 [웹서버]에 접속한 후 [웹서버]가 [웹브라우저] 쪽으로 던저주는 언어입니다.
	HTML 안에는 DB  연동 결과물이 내재되어 있다.
●HTML 은 프로그램은 언어는 아니다. 로직 같은 형식이 없습니다.
●<주의>개발자는 [입력양식] 태그를 무지 많이 조심해야한다. 오라클과 연결되고 Spring 하고도 연결되어 있다.


------------------------------------------------------------------------
■2■ HTML 차원에서 태그 (tag)가 무엇인가?
------------------------------------------------------------------------
●HTML 안에서 문자열, 이미지, 동영상, 입력양식 등을 출력하기 위해 사용되는 형식을 말한다.
------------------------------------
●태그 (tag)의 형식1
------------------------------------
	--------------------------
	<태그명  속성명="속성값">
		컨텐츠
	</태그명>
	--------------------------
	▶<태그명> 과 </태그명> 사이의 컨텐츠가 태그 기능의 영향을 받는다.
	--------------------------
	<예> <font color="blue">Bigdata</font>     => 문자열 Bigdata 가 파랑색으로 웹브라우저에 출력됨.
	<예> <b>Bigdata</b>                        => 문자열 Bigdata 가 두꺼워져 웹브라우저에 출력됨.
	<예> <center>Bigdata</center>              => 문자열 Bigdata 가 웹브라우저의 중앙위치로 출력됨.
------------------------------------
●태그 (tag)의 형식2
------------------------------------
	<태그명  속성명="속성값"/>
	--------------------------
	<예> <hr>     =>수평선 출력됨
	<예> <input type="text" name="uid" size="10"> => 한줄의 문자열을  입력하는 입력양식 출력

------------------------------------
	▶태그명 --> 태그 종류명을 말한다.
	▶속성명 --> 태그의 성질을 말한다. 
	▶속성값 --> 속성명에 대한 값을 말한다.
	▶속성명="속성값" 은 0개 이상 나올 수 있다.
	▶속성값은 " 또는 ' 로 감싼다. 만약 속성값 중간에 공백이 없으면 " 또는 '는 생략할 수 있으나 되도록 사용한다.
	▶컨텐츠에는 문자열 또는 또 다른 태그가 나올 수 있다.
	▶태그명, 속성명은 대문자 또는 소문자를 써도 된다. 되도록 소문자를 사용한다.

■9■ 웹 개발 시 [웹디자이너], [웹 퍼블리셔], [웹 프로그래머]의 역할은?
------------------------------------------------------------------------
	웹 디자이너
		포토샵, 일러스트레이터를 사용하여 웹 화면에 출력되는 이미지의 그래픽 작업을 한다.
	웹 퍼블리셔
		HTML, CSS Javascript를 사용하여 웹 하면의 UI(User Interface) 제작 작업을 한다.
		<참고> Javascript는 프로그래밍 언어의 한 종류이므로 [웹 퍼블리셔]는 프로그래머로서의 역량이 더 많이 필요하다고 볼 수도 있다.
	웹 프로그래머
		서버 쪽 언어(JSP, Java, PHP, ASP, .NET)을 사용하여 UI로부터 또는 UI로의 데이터의 이동, DB 연동 작업을 한다.
		복잡하고 어려운 Javascript 작업을 하는 경우도 있다.

■13■ a 태그?

	[문자열]이나 [이미지 태그]을 감싸고 [클릭]등과 같은 이벤트를 발생하면 다음의 기능을 구현하는 태그이다.
	[페이지 이동]
	[같은 문서내 특정위치 이동]
	[파일 다운로드]
	[이미지 출력]
	[음악, 동영상 실행]
	[자바스크립트 코드 실행]
	[MS 아웃룩 실행] 등
	------------------------------------
	<참고> 기본적으로 문자열에 마우스를 대면 자동으로 클릭 가능한 손 모양이 보인다.
	------------------------------------
	<참고> 이벤트 *** 이벤트가 실행될 때 Javascript 코드가 실행 된다. 그래서 개발자에겐 중요하다.
		- 웹상에서 이벤트란 웹브라우저에서 발생하는 모든 움직임을 말한다.
		- 주로 마우스나 키보드로 발생하지만 다른 것에 의해서도 발생한다.
			예) html 문서가 웹브라우저에 로딩되는 것, html 문서가 웹브라우저에 언로딩 되는 것(사라지는 것)
		-이벤트의 종류
			load, unload, click, dbclick, mouseover, mouseout, mousedown, mousedragged, keydown
			keyup, keypress, focus, blur, change 등등
------------------------------------

	<주의> 웹에서 열어볼 수 있는 파일은 [파일 다운로드 대화상자]가 뜨지 않고 새 화면을 열고 실행한다.

	------------------------------------
	<div style="속성1:속성값1:~;">
	------------------------------------
	style 속성
		position:static;	=> div 태그의 현재 코딩 위치에서만 존재함, 디폴트 설정이다. 
		position:relative;	=> div 태그의 현재 코딩 위치를 0, 0으로기준 삼고 x축, y축으로 이동하여 다른 컨텐츠 위에 위치한다.
		position:absolute;	=> 화면 왼쪽 모서리를 0, 0으로 기준 삼고 x축 y축으로 이동하여 다른 컨텐츠 위에 위치한다.
	
		left:화면왼쪽여백; => 웹 브라우저 왼쪽 테두리에서 div태그 영역(div 태그)까지의 가로 위치를 나타냄
					position:absolute; 또는 position:relative;가 나와야 한다.
		top:화면위쪽여백; => 웹 브라우저 위쪽 테두리에서 div태그 영역(div 태그)까지의 세로 위치를 나타냄
					position:absolute; 또는 position:relative;가 나와야 한다.

		padding:상여백 우여백 하여백 좌여백;  
				  => div태그 영역 내부의 상,우,하,좌 여백
				  => 상, 우, 하, 좌 여백이 같은 값이면 padding:여백; 으로 사용해도 된다.
				  => 상, 우, 하, 좌 여백을 따로 주고 싶으면 padding-top:여백, padding-bottom, padding-left
		
		width: div태그의 좌우 영역 너비
		height: div태그의 위 아래 영역 너비

		boder : 테두리선두께 선모양 색상 => div 태그 영역의 테두리선두께, 선무양, 색상 설정.
			예) border:2px solid red

		background-color:div 태그영역색상

		z-index:우선순위번호;   =>div 태그 영역이 겹쳤을 때 우선순위를 지정, 번호가 클수록 상위

	visibility: 보임여부설정		=> div 태그 영역 보임여부를 설정, 보임여부 설정값은 visible(보임), hidden(안보임)
					<주의> hidden 설정 시 안보이고 차지하는 공간은 남음.
	display; 보임여부 설정;		=> div 태그 영역 보임여부를 설정, 보임여부 설정값은 block(보임), none(안보임)
					<주의> none 설정 시 안보이고 차지하는 공간도 없음.

■15■ table 태그, tr 태그, td 태그가 뭐 할 때 사용하나?
------------------------------------------------------------------------
	문자열, 또는 이미지를 [행]과 [열]로 표시하는 태그이다.
	컨텐츠의 배치를 관리하는 태그이다.
	------------------------------------
	n행 m열의 행과 열을 만들려면 아래 형식을 가진다.
	<table>
		<caption>표제목</caption>

		<thead><tr><th>헤더1</th><th>헤더m</th></tr></thead>
	
	<tbody>
		<tr><td>데이터1</td>~<td>데이터1m</td></tr>
		<tr><td>데이터1</td>~<td>데이터2m</td></tr>
		<tr><td>데이터1</td>~<td>데이터3m</td></tr>
		....
		<tr></tr>
	</tbody>
	</table>
	------------------------------------
	<tr>의 개수가 전체 행의 개수다.
	한 행의 <td> 또는 <th>의 개수가 열의 개수이다.
	<thead><tbody>가 나온다는 것은 j쿼리가 나온다는 뜻이기도 하다.
	<caption>태그는 생략이 가능하다.
	<thead><tbody>는 생략 가능하다. 그러나 Jquery에서 테이블을 컨트롤 할 때는 유용하게 사용된다.
	</td></th></tr>는 생략 가능하나 영역을 나타내므로 되도록 쓴다.
		------------------------------------
		table 태그의 속성
		------------------------------------
		border	     -> 표 전체의 선 두께값 설정. border을 생략하면 border="0"이 기본이다.
		bordercolor  ->	표 전체의 테두리 색 설정	
		width	     -> 표 전체의 너비 설정
		height	     -> 표 전체의 높이 설정
		cellpadding  -> 각 셀 안의 문자와 경계선 사이 여백 설정
		cellspacing  -> 각 셀 과 셀 사이의 튀어나온 부분의 너비. cellspacing을 생략하면 cellspacing="2"가 기본이다.
		align	     -> 표 전체의 위치(왼쪽, 가운데, 오른쪽) 설정
		bgcolor	     -> 표 전체의 배경색 설정
		background   -> 표 전체의 배경 이미지 설정
		style	     -> 1개 이상의 CSS 설정할 때 사용한다.
				<table style="border-collapse:collapse"> 테이블의 입체 경계선을 단일 선으로 만든다.
		------------------------------------
		tr 태그의 속성
		------------------------------------
		bgcolor	     -> 한 행의 배경색 설정
		style	     -> 1개 이상의 SCC 설정할 때 사용한다.
				<tr style="cursor:pointer"> 행에 마우스를 대면 손모양이 보인다.
		on 이벤트명  -> 이벤트가 발생하면 원하는 자바스크립트 코딩을 실행한다.
		------------------------------------
		td, th 태그의 속성
		------------------------------------
		width	     -> 셀의 너비 설정
		height	     -> 셀의 높이 설정
		align	     -> 셀 안의 좌우중의 위치 설정
		valign	     -> 셀 안의 상중하의 위치 설정
		bgcolor	     -> 셀의 배경색 설정
		backgroung   -> 셀의 배경 이미지 설정
		colspan	     -> 지정한 개수의 열만큼 좌우셀을 합친다
		rowspan	     -> 위아래 셀을 합친다

■16■ 입력 양식 관련 태그의 종류
------------------------------------------------------------------------
입력 양식 관련 태그는 [데이터]를 [입력] 또는 [선택]할 수 있는 태그 또는 그와 관련된 태그를 말한다.
입력 양식 관련 태그에는 입력 또는 선택한 데이터는 서버의 웹프로그래밍 언어로 전달되고 최종 DB(=오라클)에 저장된다.
	------------------------------------
	입력 양식 관련 태그의 종류
	------------------------------------
	form
		여러 [입력 양식]을 1개의 그룹으로 묶는 태그다. 택배 상자와 비슷한 역할
		서버의 URL주소, 데이터 전송 방법 등을 설정할 수 있다.
		실질적인 입력양식은 아니나 중요한 태그다.
		------------------------------------
		form 태그의 속성
		------------------------------------
			name   -> form 태그의 이름을 설정하는 속성명. form 태그를 구별 또는 지칭하기 위해 주는 속성.
			action -> form 태그 안의 입력양식에 입력(선택)된 데이터를 보낼 [서버 쪽 URL 주소]를 설정하는 속성.
			method -> 입력(선택) 데이터를 서버로 보내는 방법을 설정한다.
				예) method="get"  => 입력(선택) 데이터가 URL뒤에 붙어 전송.
							보안 수준이 낮은 데이터 전송 시 사용.
							이 방식으로 데이터를 전송할 때 URL 주소에 보낼 전송 데이터를 삽입만 하면 되므로 코딩이 짧아진다.
							입력 양식이 필요 없다.
							method 속성 설정이 생략되면 method="get"가 디폴트이다.
				    method="post" => 입력(선택) 데이터가 안보이게 전송.
							보안 수준이 높은 데이터 전송 시 사용.
							이 방식으로 데이터를 전송할 때 입력양식에 데이터를 삽입해야 하므로 코딩이 길어진다.
							입력 양식이 필요하다.
			enctype -> 전송할 데이터의 형식을 설정. 문자 또는 파일을 보낼 때 설정 해준다.
				   입력 양식 중에 파일업로드가 있으면 enctype 속성 값을 준다.
				   enctype="multipart/form-data"
				   <주의>생략하면 파일은 전송되지 않고 파일의 이름만 전송된다.
			target  -> form 태그 안의 입력양식에 입력(선택)된 데이터를 보낼 때 어느 화면을 이용해 보낼지 설정한다.
				   target 속성이 생략되면 form 태그가 있는 현재 화면을 이용하여 웹서버로 전송한다.
	------------------------------------
	input
		데이터를 입력 또는 [선택](=체크)할 수 있는 [입력 양식]을 출력하는 태그다.
		------------------------------------
		input 태그의 속성
		------------------------------------
			type="text"     -> 한 줄의 [문자열]을 입력받는 입력양식을 설정한다.
			type="password" -> 한 줄의 [암호문자열]을 입력받는 입력양식을 설정한다.
		*	type="checkbox" -> [체크박스] 양식 설정. 다중 체크 가능. 관련된 모든 프로그래밍이 복잡해진다. DB의 테이블 구조도 복잡해진다.
			type="radio"	-> [라디오버튼] 입력 양식 설정한다. 다중 체크 불가능.
		*	type="file"	-> [파일박스] 양식 설정. 파일 업로드에 사용. 서버 프로그래밍이 복잡해진다.
					   파일 업로드 사용 시 보안성에 문제가 생길 가능성이 있다. 해커 침투 경로이기도 하다.
					   이 입력 양식이 사용됐으면 enctype="multipart/form-data" 가 있어야 한다.
		*	type="hidden"	-> 화면에 보이지 않고 값만 저장하는 양식 설정
					   화면에는 노출하지 않고 서버로는 데이터를 보내려 할 때 사용한다.
					   검색화면에서 자주 사용된다.
					   <팁>개발과정에서는 type="text"로 개발한 후 완료 시에 type"hidden"로 변경한다.
					   오라클[PK] 값이 저장되기도 한다.
			------------------------------------
			type="button"	-> [버튼] 약식 설정. 사용자 정의 버튼. 버튼을 클리하면 벌어지는 일은 개발자가 코딩해야 한다.
			type="submit"	-> [서버 전송 버튼] 양식 설정. 같은 폼 소속의 데이터를 서버쪽으로 보내는 전용 버튼이다.
			type="reset"	-> [초기화 버튼] 양식 설정은 폼 소속의 입력 양식 데이터를 초기화하는 전용 버튼이다.
			위 세가지는 입력양식에 포함은 되나 value값은 서버로 전송되지는 않는다.DB연동과는 관련이 없다.
			------------------------------------
		*	name=""		-> 입력 양식 이름을 지정. 주로 [오라클]의 [컬럼명]과 일치 시킨다.
		*	value		-> 입력 또는 선택한 값이 저장되는 속성. 서버의 웹 프로그래밍 언어로 전달되고 최종 DB의 테이블에 저장.
					   <주의> type="button", type="submit", type="reset" 의 경우 버튼에 삽입되는 단순 문자열 설정이고, 서버의 웹 프로그래밍 언어로 전달되지 않는다.
			size		-> 입력 양식의 좌우 길이를 설정. <주의> type="text", type="password"에서는 좌우 길이지만 select 태그 안에서는 펼쳐 놓은 목록 개수를 말한다.
			maxlength	-> 키보드로 입력 가능한 문자열의 개수.
					   <주의> type="text", type="password" 에만 설정 가능
					   <주의> 입력한 데이터가 한글일 경우 브라우저마다 maxlength 속성이 안 먹힐 수 있다.
						이럴 경우 입력 가능한 문자열의 개수는 자바스크립트에서 체크한다.
			checked		-> type="checkbox", type="radio"인 양식에서 삽입되어  디폴트 선택(=체크) 하게 만든다.
						예) <input type="checkbox" name="lunar_or_solar" value="lunar" checked> 양력
	------------------------------------
	select
		1개 이상의 목록 중 하나를 선택하는 [목록상자]를 출력하는 태그다.
		목록을 감추고 클릭하면 목록을 볼 수 있어 공간을 작게 차지한다.
		------------------------------------
		select 태그의 속성
			name=""		-> 입력 양식 이름을 지정. 주로 [오라클]의 [컬럼명]과 일치 시킨다.
			size		-> select 태그 안에서 펼쳐놓는 목록개수
			multiple	-> 다중 선택 여부를 지정. multiple 설정 시 ctl키를 누르고 다중 선택 가능.
					   주로 관리자 모드에서 사용한다.
	------------------------------------
	option
		select 태그 내부에서 사용되어 [목록상자] 내부의 목록을 출력하는 태그다.
		------------------------------------
		option 태그의 속성
		------------------------------------
			value		-> DB에 저장될 실제값 저장. 예) <option value="1">JSP</option>
			selected	-> 목록을 선택하게 함(디폴트 목록선택).	 예) <option value="JSP" selected>JSP</option>
	------------------------------------
	textarea
		다중 행으로 문자열을 입력할 수 있는 [입력 양식]을 출력하는 태그다.
			name		-> 입력 양식 이름을 지정 속성이다. 주ㅡ로 [오라클]의 [컬럼명]과 일치 시킨다.
			rows		-> 보이는 행의 개수 설정
			cols		-> 보이는 열의 개수 설정
	------------------------------------

■17■ HTML에서 이벤트가 무엇인가?
------------------------------------------------------------------------
웹 브라우저에서 발생하는 모든 움직임을 말한다.
이벤트는 주로 유저에 의해 발생되고 이벤트는 웹서버에게 일종의 명령을 내리는 것과 동일하다.
이벤트는 웹브라우저와 웹서버간의 소통역할을 한다.
발생하는 이벤트를 웹브라우저가 감지하고 있다.
	------------------------------------
	웹브라우저가 감지할 수 있는 이벤트의 종류
	------------------------------------
	click 이벤트        -> 마우스 왼쪽 버튼을 클릭하는 행위, 가장 많이 사용하는 이벤트.
	mouseover 이벤트    -> 글씨나 이미지에 마우스를 갖다 대는 행위이다.
	mouseout 이벤트	    -> 글씨나 이미지에 마우스를 갖다 댄 후 때는 행위이다.
	mousedown 이벤트	    -> 마우스에 달려 있는 임의 버튼을 클릭하는 행위이다.
	mousedragged 이벤트 -> 마우스 버튼을 누르고 드래그하는 행위이다.
	keydown 이벤트	    -> 키보드를 누르는 행위이다. 키보드를 누르고 있을 때 한번만 이벤트가 발생한다.
	keypress 이벤트	    -> 키보드를 누르는 행위이다. 키보드를 누르고 있을 때 계속 이벤트가 발생한다.
	keyup 이벤트	    -> 키보드를 누른 후 때는 행위.
	select 이벤트	    -> 문자열을 드래그해서 선택하는 행위이다.
	change 이벤트	    -> select 태그라면 목록을 바꿔 선택하는 행위이다.
			       <input type="text"> 또는 <input type="password" ~> <textarea ~>에서는 입력내용이 변동된 후 커서가 빠져나올 때 발생하는 이벤트
			       <input type="checkbox"> 또는 <input type="radio"~> 에서는 체크가 들어가거나 체크가 빠지는 이벤트이다.
	load 이벤트	    -> <body> 태그 안의 모든 내용을 읽어 화면에 출력(=로딩)한 후 발생되는 이벤트이다.
	unload 이벤트	    -> 현재 보여지는 웹페이지가 웹브라우저 화면에서 사라질 때 발생하는 이벤트
	기타 등등
